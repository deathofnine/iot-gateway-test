## 缺陷严重程度定义

| 等级 | 说明 |
|------|------|
| **致命（Fatal）** | 系统崩溃、主要功能完全失效 |
| **严重（Critical）** | 主要功能部分失效，影响核心业务 |
| **一般（Major）** | 功能缺陷，但有替代方案 |
| **轻微（Minor）** | 界面显示问题、文档错误等 |

---

## 缺陷状态定义

| 状态 | 说明 |
|------|------|
| **新建（New）** | 刚发现，未确认 |
| **确认（Confirmed）** | 确认是缺陷，等待修复 |
| **分配（Assigned）** | 已分配给开发人员修复 |
| **已修复（Fixed）** | 开发完成，等待验证 |
| **关闭（Closed）** | 测试验证通过，缺陷关闭 |
| **重新打开（Reopened）** | 修复后验证失败 |

---

## 缺陷报告模板

| 字段 | 说明 |
|------|------|
| 缺陷ID | 格式：BUG_XXX（自增编号） |
| 发现日期 | YYYY-MM-DD 格式 |
| 测试用例ID | 发现该缺陷的测试用例编号 |
| 缺陷描述 | 详细描述缺陷现象 |
| 严重程度 | 致命/严重/一般/轻微 |
| 状态 | 当前缺陷状态 |
| 备注 | 其他补充说明 |
